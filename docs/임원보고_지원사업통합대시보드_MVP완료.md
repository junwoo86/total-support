1. **자동 수집 파이프라인** — 3 사이트 매일 새벽 04:00 KST 자동, 사용자 "▶ 지금 실행" 트리거도 가능

## 1. 배경 · 왜 이 시스템을 만들었는가

### 1.1 현재 운영 상태 (시스템 도입 전)

알로스타는 시리즈 A 7년차로, 정부·지자체 지원사업을 통한 R&D 자금·사업화 비용·IR 매칭·인프라 확보가 **회사 성장의 핵심 통로 중 하나**입니다.

지원사업 정보는 다음 3 사이트에 사실상 모두 모입니다.

| 사이트 | 성격 | 일 평균 신규 |
|---|---|---|
| 기업마당 (bizinfo.go.kr) | 일반 지원사업 (중기부·지자체·산하기관) | ~150건 |
| IRIS (iris.go.kr) | 범부처 R&D 과제 | ~15건 |
| SBA (sba.seoul.kr) | 서울경제진흥원 (서울시 한정) | ~10건 |

그러나 **각 사이트는 UI/검색/접근 방식이 모두 달라**, 사람이 매일 3 사이트를 개별 방문해 신규 공고를 확인하는 구조였습니다.

**현장에서 일어나고 있던 일**:
- 대표를 포함한 임원·실무 담당자 여러 명이 **비정기적으로** 검토 (전담자 없음).
- 최근 회사가 다루는 업무량 자체가 늘어나면서, 지원사업 모니터링이 **후순위로 밀림**.
- 결과: **작년/재작년 대비 지원사업 신청 건수가 급감**.
- 더 큰 문제는 단순 신청 감소가 아니라, **신청했어야 할 적합 사업을 마감 후 또는 마감 직전에 발견하는 사례가 다수 발견**되었다는 점.

### 1.2 기회비용 — 보이지 않는 손실

지원사업 1건의 평균 수혜 규모는 영역에 따라 다르지만, 통상:
- R&D 과제 1건 = 수천만 ~ 수억 원
- 사업화/마케팅 지원 = 수백만 ~ 수천만 원
- 액셀러레이팅/IR 매칭 = 직접 자금 + 후속 투자 연결의 잠재 가치

알로스타가 명시적으로 자격 갖춘 후보군 (벤처기업·이노비즈·바이오/디지털헬스/AI 분야)이 매일 3 사이트에 노출되고 있는데, 이를 **단지 정보 탐색 비용 때문에 놓치고 있다**면 그 자체가 회사가 받을 자원을 받지 못하고 있는 것입니다.

> **요컨대 "사람이 매일 3 사이트를 보지 않아서 회사가 받을 자금을 못 받는 구조"가 만들어져 있었습니다.**

## 2. 가설 · 무엇을 해결하면 풀리는가

### 2.1 핵심 가설

> **3 사이트 자동 수집 + AI의 회사 지침 기반 사전 평가 + 검토 상태 관리 워크플로우** 의 세 가지를 갖추면, 사람의 검토 부하를 **90% 절감**하면서 **마감 누락을 0**으로 만들 수 있다.

### 2.2 가설을 뒷받침하는 논리

| 단계 | 정량 가정 |
|---|---|
| 매일 3 사이트 신규 공고 총량 | ~180건/일 |
| 알로스타 분야와 키워드가 1개라도 매칭되는 건 | ~30~40건/일 (15~20%) |
| AI 적합도 ≥ 80 (즉시 검토 가치) | ~5~15건/일 |
| 사람이 결국 실제 검토해야 할 건 | **5~15건/일** |
| → 사람의 1일 검토 시간 | **5~10분** (기존 1.5~2 시간 대비 −90%) |

### 2.3 검증 방법

- **시간 절감**: 사용자의 일일 검토 행위(상태 변경 이벤트)를 시스템 로그(`tb_grant_system_logs`)에서 측정 → 평균 PATCH 1건당 소요 시간 추정.
- **누락 0건**: 운영 30일 후 외부 별도 채널(공식 메일링/유관기관 안내 등)에서 받은 공고와 시스템 수집분을 대조 → 시스템에 누락된 건이 있는지 확인.
- **신청 건수 회복**: 운영 전/후 6개월 신청 건수 비교.

### 3.4 핵심 화면 ② — AI 회사 지침 (분야·키워드 관리 탭)

> **이 시스템의 차별점 중 하나.** AI가 일반적인 분야 키워드만 보고 평가하면 노이즈가 큰데, 우리는 **회사 정체성과 전략을 자연어로 적은 지침**을 AI에게 시스템 프롬프트로 줍니다.

지침 예시 (현재 v2 · 2,312자):
> 바이오(기능의학) 기반 **대사체 분석** 키트와 **헬스케어 솔루션**...
> ⚠️ 우리는 의료 진단(diagnosis) 영역을 침범하지 않기 위해 의도적으로 '분석(analysis)'이라는 표현을 사용합니다. "진단키트" 같은 분류로 단순 분류된 사업이라도 본문이 실제로는 대사체·바이오마커·헬스케어·웰니스 데이터 분석에 가까우면 높게 평가하세요.
> 지역 한정 사업은 '서울' 또는 '광주(전라도)' 만 포함...

지침을 수정하면 미검토 + 검토 필요 공고가 자동 재평가됩니다 (검토 시작한 행의 과거 점수는 보존). **추가 코드 배포 없이 AI 평가 기준을 회사 전략 변화에 맞춰 조정 가능**.

### 3.5 핵심 기능 ③ — 검토 워크플로우 (4 상태)

```
[미검토 UNREVIEWED]  ──► [검토 필요 NEEDS_REVIEW]  ──► [지원 진행 IN_PROGRESS]
                  └────► [제외 EXCLUDED]
```

- **일괄 처리**: 체크박스로 다건 선택 → 한 번에 상태 변경
- **상태별 카운트**: 탭 옆 + 각 칩 옆에 정확한 건수 표시 (PATCH 즉시 갱신)
- **만료 자동 숨김**: 검토 필요 / 지원 진행에서 마감 지난 공고는 기본 숨김 (체크박스로 해제 가능)

```
상 (80%↑)   ████████ 77건       ← 즉시 검토 가치 (전체의 8.5%)
중상 (60%↑) ████ 40건            ← 검토 권장 (4.4%)
중 (40%↑)   █ 11건                ← 인접 영역
하 (40%↓)   ████████████████████████████████████████████████████ 725건
                                  ← 무관 또는 명시적 제외 (80%)
평가 안 함  ███ 54건               ← 본문 너무 짧음 등
분석 실패   1건                  ← Gemini 3회 재시도 모두 실패 (UI 최상단 노출)
```

**핵심 함의**:
- 5일간 사람이 보아야 할 후보가 908건 → **117건** (상+중상)
- 그중 우선순위 후보는 **77건** (상)
- 즉 **검토 부하 90% 절감 가설은 데이터로 검증됨** — 908건 중 8.5% 만 본격 검토 대상.

## 6. 기대 효과

### 6.1 단기 (운영 시작 ~ 1개월)

1. **임원 검토 부하 90% 절감** — 매일 5~10분으로 압축
2. **마감 누락 0건** — 매일 자동 수집 + AI 사전 점수화로 D-Day 임박 사업도 빠짐없이 수면 위로 노출
3. **신청 의사결정 속도 향상** — AI 가 적합 사유를 미리 작성하여, 사람이 "맞다/아니다" 판단만 하면 됨

### 6.2 중기 (운영 1~6개월)

1. **신청 건수 회복** — 작년/재작년 대비 급감했던 신청 건수가 정상 수준 이상으로 회복
2. **회사 지침 정교화** — AI 평가 결과를 보며 회사 지침 v2 → v3 → v4 로 누적 개선 (이미 append-only 히스토리 구조)
3. **분야 키워드 보강** — 데이터 기반으로 매칭 누락 패턴 파악 (예: 웰니스 키워드 1건만 매칭됨 → 보강 후보)

### 6.3 잠재 효과 (정량화 어려움)

- **놓친 R&D 자금 회복** — 채택 1건 시 수천만~수억 원, 누락 방지의 ROI 는 시스템 운영 비용의 수십~수백 배.
- **임원 의사결정 시간을 본업으로 재배치** — 매일 1.5~2 시간을 다른 전략 업무에 투입.
- **회사 전략과 지원사업 매칭의 명문화** — 회사 지침이 AI 의 입력이 되면서, 회사 전략 자체가 텍스트로 문서화되는 부수 효과.

## 7. 향후 로드맵 (다음 단계 제안)

### 7.1 다음 마일스톤: **AI 자동 신청서 초안 작성**

현재까지는 "공고를 찾고 → 적합도를 평가" 까지였습니다. **다음 단계는 "신청하기로 결정된 건에 대해 AI 가 연구/사업 주제를 추천하고 신청서 개요를 자동 작성"** 하는 것입니다.

```
[현재 MVP]
공고 자동 수집
   ↓
AI 적합도 평가 (점수 + 사유)
   ↓
사람의 검토 (적합 / 제외)
   ↓
사람이 신청서 직접 작성 ◄ 여기가 다음 자동화 대상
```

```
[다음 마일스톤]
공고 자동 수집
   ↓
AI 적합도 평가
   ↓
사람의 검토 (적합 / 제외)
   ↓
"지원 진행" 상태 변경 시 → AI 가
   1. 회사 강점 / 키트 기술 / 데이터 자산을 공고와 매핑
   2. 연구·사업 주제 후보 3개 제안
   3. 신청서 개요(요약 / 추진 일정 / 예산 / 기대성과) 자동 초안 작성
   ↓
사람이 초안 검토 + 보강 → 제출
```

**기술적 실현성**: 현재 사용 중인 Vertex AI Gemini + 회사 지침 (이미 구축)에 신청서 양식 + 회사 강점 자료를 더 추가하면 됨. 추가 인프라 없음.

**기대 효과**: 신청서 작성에 들어가는 시간을 추가로 50~70% 절감 가능 (사람은 초안 검토와 도메인 보강만).

### 7.2 그 외 검토 가능한 확장

- **다른 정보원 추가** — K-Startup, 창업진흥원, 산업부 산하기관 등 추가 사이트 통합
- **채택 후 사후관리** — 채택된 사업의 보고서/정산 마감일 알림, 후속 사업 매칭
- **유사 사업 회상** — 과거 채택된 사업과 유사한 신규 공고가 뜨면 우선 알림
- **이메일/Slack 알림** — D-7 임박 + 적합도 상 인 공고가 신규 발견되면 자동 알림

Design System Specification

Generic Modern Web Design System

범용 웹 페이지 및 제품군 페이지 개발에 사용할 수 있는 모듈형 디자인 시스템 명세. 스타크한 모노크롬 베이스 위에 카테고리별 채도 컬러 카드를 얹는 이중 정체성 패턴을 따른다.

1. Overview

이 디자인 시스템은 모노크롬 베이스 + 채도 높은 카테고리 카드라는 이중 정체성으로 구성된다. 마케팅 표면과 플랫폼 페이지는 화이트 캔버스와 거의 블랙에 가까운 타이포그래피로 앵커되며, 보이스는 자신감 있고 기술적이며 에디토리얼하다. 그러나 각 제품/기능 카테고리는 고유 그라디언트 컬러 카드로 개성을 가진다 — 카테고리 A는 코랄, 카테고리 B는 마젠타, 카테고리 C는 블루, 카테고리 D는 퍼플처럼.

단일 변동성 산세리프(DM Sans 권장, Inter 폴백)가 80px 헤로 디스플레이부터 12px 마이크로 라벨까지 모든 표면을 앵커한다. 버튼은 보편적으로 pill 형태(rounded-full)이며 명확한 2계층 체계를 따른다: 블랙 pill 프라이머리(지배적 CTA)와 아웃라인 pill 세컨더리. 카드는 두 패밀리로 분리된다: 채도 그라디언트 쇼케이스 카드(32px 코너)와 조용한 화이트 문서 카드(16px 코너).

핵심 특성

  • 스타크한 모노크롬 팔레트 — 블랙({colors.primary})과 화이트({colors.canvas}) — 채도 그라디언트 카테고리 카드로만 깨짐
  • 카테고리별 컬러 인코딩: 각 제품/섹션 라인이 고유 브랜드 컬러를 가짐
  • 시스템 전체에 단일 타입페이스 — DM Sans 또는 Inter
  • 모든 버튼, pill 탭, 뱃지에 {rounded.full} 적용
  • 헤로 타이포그래피는 1.10 라인하이트 + -2px 자간으로 임팩트 확보
  • 문서 표면은 3컬럼 레이아웃: 좌측 사이드바 / 중앙 본문 / 우측 TOC
  • 상단 네비게이션 위에 블랙 프로모 배너 사용 가능 (시간 제한 이벤트용)

3. Typography

3.1 Font Family

DM Sans(주력): 변동성 산세리프. 모든 표면, 모든 역할에 사용. 폴백: Inter, Helvetica Neue, Helvetica, Arial.

이 타입페이스는 80px 헤로 디스플레이(-2px 자간으로 매거진 급 타이트함 구현)부터 12px 마이크로 라벨까지 깔끔하게 스케일링된다. 이탤릭 변형은 사용하지 않으며 — 강조는 웨이트(500/600/700)로 처리한다.

3.3 Principles

  • 타이트한 헤로 leading(1.10)과 공격적인 음수 자간을 디스플레이 사이즈에 적용 — 매거진급 타이포그래피 구현
  • 여유로운 본문 leading(1.50) — 긴 문서 가독성 확보; 캡션은 1.70까지 확장
  • 웨이트 디시플린: 400(본문), 500(중간 강조), 600(헤딩/버튼), 700(인라인 강조). 더 무거운 웨이트는 사용하지 않음
  • 단일 타입페이스 전략 — 다른 산세리프와 혼용 금지. 코드 샘플은 시스템 모노스페이스 폴백 사용

4. Layout

4.1 Spacing System

  • Base unit: 4px (8px 주력 증가 단위)
  • Tokens: {spacing.xxs} (4px) · {spacing.xs} (8px) · {spacing.sm} (12px) · {spacing.md} (16px) · {spacing.lg} (20px) · {spacing.xl} (24px) · {spacing.xxl} (32px) · {spacing.xxxl} (40px) · {spacing.section-sm} (48px) · {spacing.section} (64px) · {spacing.section-lg} (80px) · {spacing.hero} (96px)
  • 섹션 리듬: 마케팅 페이지는 어보브폴드 {spacing.hero}(96px), 이후 {spacing.section-lg}(80px); 문서는 {spacing.section}(64px)로 타이트하게; 테이블 로우는 {spacing.md}(16px)로 압축
  • 카드 내부 패딩: 채도 카드는 {spacing.xxl}(32px); 문서 카드는 {spacing.lg}–{spacing.xl}(20–24px); 프로모 스트립은 {spacing.section}(64px)까지 확장

4.2 Grid & Container

  • 마케팅 페이지: 1280px 최대 너비, 32px 거터
  • 홈페이지 카테고리 매트릭스: 32px 라운드 그라디언트 카드 4컬럼 행, 각 카드 ~280–320px 너비
  • 제품 매트릭스(아래): 16px 라운드 화이트 카드 4컬럼 그리드
  • 문서 표면: 3컬럼 — 좌측 사이드바(~220px) / 중앙 본문(~720px 최대) / 우측 TOC(~180px). 1024px 이하에서 드로어로 접힘
  • 가격/플랜 페이지: 2컬럼 탭 위에 3컬럼 티어 카드 그리드

4.3 Whitespace Philosophy

마케팅 페이지는 제품 사진과 컬러 카드에 넉넉한 호흡을 제공한다 — {spacing.hero}(96px)는 80px 헤로 디스플레이를 위한 시각적 산소를 만든다. 문서 내부에서는 화이트스페이스를 급격히 타이트하게 가져간다: 섹션 갭은 {spacing.xxl}(32px)로 떨어지고, 테이블 로우는 {spacing.md}(16px)로 압축되며, 사이드바 네비는 {spacing.xs}(8px) 수직 리듬을 사용한다.

5. Elevation & Depth

시스템은 주로 플랫하다. Elevation은 고정 패널, 드롭다운, 드물게 떠 있는 CTA에만 사용된다.

0 (flat)그림자 없음; {colors.hairline} 보더기본 카드, 테이블 로우, 폼 입력
1 (subtle)rgba(0,0,0,0.04) 0 1px 2px 0호버 엘리베이션 타일

5.1 Decorative Depth

  • 채도 그라디언트 카드는 내부 래디얼 그라디언트와 실루엣 이미지로 자체 깊이 표현 — 그림자 불필요
  • 퍼플 테마 카드 아래에 브랜드 톤 그림자(rgba(44, 30, 116, 0.16) 0 0 15px)로 미묘한 앰비언트 리프트 부여 가능

  • 채도 카드는 32px 코너 — 화이트 카드(16px)와 의도적 대비. 이 두 배 라디우스가 "이건 피처드 모먼트다"라는 시각 시그니처
  • 아바타 원은 {rounded.full} — 완벽한 원

7. Components

No-hover 정책에 따라 호버 상태는 명세하지 않음. Default와 pressed/active 상태만 다룬다.

docs-prose-block

stat-row

8. Do's and Don'ts

✅ Do

  • {colors.primary}(블랙)을 지배적 CTA로 사용 — 가장 인식 가능한 인터랙티브 요소
  • 카테고리 컬러({colors.accent-coral}, {colors.accent-magenta}, {colors.accent-blue}, {colors.accent-purple})는 카테고리 정체성 모먼트에만 사용
  • {rounded.hero}(32px) 채도 카드와 {rounded.xl}(16px) 화이트 카드를 같은 뷰포트에 페어링 — 라디우스 대비가 시각 시그니처
  • 모든 버튼, pill 탭, 뱃지에 {rounded.full} 적용
  • 헤로 디스플레이는 {typography.hero-display}(80px) + -2px 자간 사용 — leading/자간 타협 금지
  • 각 제품/카테고리 라인을 별개의 컬러 정체성으로 다루기 — 자유 선택이 아닌 시스템 할당

❌ Don't

  • 카테고리 컬러를 본문 텍스트나 대형 표면에 사용하지 말 것 — 의미가 희석됨
  • 버튼 코너를 {rounded.full} 미만으로 부드럽게 만들지 말 것 — Pill은 시스템 시그니처
  • 두 번째 디스플레이 타입페이스 도입 금지 — 단일 타입페이스가 모든 역할 담당
  • 헤로 leading을 1.10 미만으로 줄이지 말 것
  • 화이트 카드에 강한 그림자 적용 금지 — flat + border가 문서 기본값
  • 표준 버튼에 그라디언트 배경 적용 금지 — 그라디언트는 카테고리 카드 정체성에만

9.2 Touch Targets

  • Pill 버튼: 데스크톱 38–40px → 모바일 44px (패딩 오버라이드)
  • 원형 아이콘 버튼: 36×36px → 모바일 44×44px
  • 폼 입력: 40px → 모바일 44px
  • 사이드바 네비: 32px → 모바일 드로어 44px

9.3 Collapsing Strategy

  • 프로모 배너: 풀폭 유지, < 480px에서 한 줄 truncate
  • 상단 네비: 1024px 이하 햄버거
  • 문서 그리드: 3컬럼 → <1024px 사이드바 드로어 → <768px 싱글 컬럼
  • 카테고리 매트릭스: 4컬럼 → <1024px 수평 스크롤 (snap 포인트 캐러셀)
  • 제품 매트릭스: 4컬럼 → 태블릿 2컬럼 → 모바일 1컬럼
  • 헤로 타이포: 80px → <1024px 56px → <768px 40px → <480px 32px
  • 스탯 스트립: 4컬럼 → <768px 2×2 → <480px 1컬럼

10. Iteration Guide

  1. 한 번에 한 컴포넌트에 집중. 시스템은 높은 내부 일관성을 갖는다.
  2. 컴포넌트명과 토큰을 직접 참조 ({colors.primary}, {component-name}-pressed, {rounded.full}) — 의역 금지
  3. 새 변형은 별도 컴포넌트 엔트리로 추가 (-pressed, -disabled, -active)
  4. 본문 기본은 {typography.body-md}, 강조 기본은 {typography.subtitle}. 헤드라인은 hero-display → display-lg → heading-lg → heading-md → heading-sm 순으로 step-down
  5. 카테고리 컬러를 카드 정체성에 한정. 표준 버튼이나 일반 표면에 카테고리 컬러가 보이면 "이 표면이 그것을 얻을 자격이 있는가?"를 자문
  6. Pill 버튼({rounded.full})은 항상; 사각 버튼은 이 언어에서 "서드파티 위젯" 신호로 읽힘

11. Known Gaps

  • 다크 모드 토큰(canvas, surface, ink, hairline)이 본 명세에 포함되지 않음 — 별도 정의 필요
  • 애니메이션/트랜지션 타이밍 미포함; 호버/포커스 트랜지션은 150–200ms ease 권장
  • 폼 유효성 성공 상태는 기본 외 명시 안 됨 — 그린 보더 + success 뱃지 표준 패턴 권장
  • 문서 내 코드 신택스 하이라이팅 팔레트 미정 — 시스템 모노스페이스 + 최소 컬러링 권장
